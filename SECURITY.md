# Security Policy

This repository primarily contains documentation, templates, and examples for operational readiness of tool-using LLM agents.

## Reporting security issues
If you believe there is a security issue in runnable code under `tools/` (if present in future), or in published artifacts that could cause unsafe reuse, please report it privately.

Contact: Rogel S.J. Corral  
Channel: use a private email channel you designate or a private GitHub message.

## Scope
- Documentation errors and conceptual issues should be filed as Issues.
- Security vulnerabilities apply only to runnable code if and when such code exists in this repository.
